# Apollo Sports Event Management System

A full-stack web application designed for **The Apollo University** to drastically simplify sports event management, student participant registrations, volunteer attendance tracking, and live scoring.

## Features
- **Student Portal**: View available games, read rules, register (capturing Department & Roll No/Batch), and track match fixtures.
- **Admin Dashboard**: Create Events & Fixtures, evaluate detailed student registration logs, securely delete dependent arrays, and publish live match scores.
- **Role-Based Operations**: Secure login flows via `Flask-Login` ensuring data integrity and safe volunteer attendance marking.

## Tech Stack
- **Backend:** Python 3, Flask, SQLAlchemy, Flask-Login, Flask-WTF
- **Database:** SQLite3 (Development Setup)
- **Frontend:** HTML5, Jinja2 Templates, Bootstrap 5

## Getting Started (For Development Team)

There are two primary ways to spin up the Apollo Sports Event system locally for development:

### Option 1: Vibe Coding via Google Antigravity
If your teammates are using Google Antigravity inside VS Code:
1. Open the project folder in VS Code.
2. Launch Google Antigravity.
3. Simply prompt: *"Initialize the Apollo Sports project, install the dependencies, and run the backend Flask server."*
4. Antigravity will automatically analyze `requirements.txt`, spin up the virtual environment, execute the `seed_db.py` initialization, and securely run the local server on `http://127.0.0.1:5000` for you!

### Option 2: Manual Setup
If you are setting up the project manually through the terminal, follow these steps:

#### Prerequisites
Make sure you have [Python 3.8+](https://www.python.org/downloads/) installed on your machine.

#### 1. Setup the Virtual Environment
To prevent dependency conflicts across your team, initialize a local virtual environment in the project directory:

**Windows:**
```powershell
python -m venv venv
.\venv\Scripts\activate
```

**Mac/Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

#### 2. Install Dependencies
With the virtual environment activated, install all the required Python packages:
```bash
pip install -r requirements.txt
```

#### 3. Initialize the Database
Before running the server for the very first time on a new computer, you must build the SQLite database tables and seed the initial developer data. Run the custom seed file:
```bash
python seed_db.py
```
*This will generate `app/sports.db` and inject the core backend architecture.*

#### 4. Boot the Server
Start the development server:
```bash
python run.py
```
Access the application locally at: **http://127.0.0.1:5000**
